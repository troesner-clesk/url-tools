Link Checker — Domain Badge Demo

This page exists to exercise every branch of the domain checker (server/utils/domain-checker.ts). Crawl this URL in the Link Checker tab with External links only disabled. The Domain column should then show a badge for each of the first two links below.

Links

  • Unregistered domain (.invalid TLD) — triggers Available. The .invalid TLD is reserved by RFC 2606 and will always return NXDOMAIN.
  • Missing subdomain on a live parent domain — triggers Subdomain missing. The leaf fails to resolve but google.com (the registrable parent) does.
  • Valid reference link — resolves cleanly, no Domain badge, Status 200.
  • Valid domain, 404 path — Status 404, but Domain column stays empty. Demonstrates that HTTP status and DNS status are independent dimensions.

Expected result

Link Status Domain badge
*.invalid ERR Available
no-such-sub-xyz987.google.com ERR Subdomain missing
example.com 200 (empty)
example.com/this-path-does-not-exist 404 (empty)
